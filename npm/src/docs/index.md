## Огляд

Інкапсулює логіку для відправки текстових повідомлень та документів через Telegram API (<https://api.telegram.org/bot>). Визначає максимальну довжину повідомлень (MAX_TELEGRAM_MSG_LENGTH) та використовує режим парсингу, встановлений у константі DEFAULT_PARSE_MODE="MarkdownV2", для коректної взаємодії з сервісом. Функціонал спирається на конфігурацію, завантажену з `res.json`. Взаємодія з мережею реалізована з механізмами fail-safe, що забезпечує перехоплення помилок; замість викиду винятків при певних збоях повертається порожнє значення (наприклад, null).

## Поведінка

Поведінка
MAX_TELEGRAM_MSG_LENGTH визначає максимальну довжину повідомлення, яке може бути відправлено в Telegram.
DEFAULT_PARSE_MODE встановлює стандартний режим парсингу для повідомлень Telegram.
escapeMarkdownV2 екранує спеціальні символи MarkdownV2 у текстових даних, які передаються в Telegram.
sendMessage відправляє текстове повідомлення до користувача через API Telegram.
sendDocument відправляє документ до користувача через API Telegram.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
